# 知音 Web Demo UI 参考说明

更新时间：2026-05-16

## 1. UI 目标

知音必须被设计成抖音信息流里的子功能，而不是独立 AI 工具、旅行攻略页或外部 Web App。

视觉优先级：

1. 先像抖音视频信息流。
2. 再自然长出知音入口。
3. 半屏卡和延展页沿用抖音弹层 / 面板语法。
4. 知音品牌感只做小面积点缀，不破坏信息流原生感。

## 2. 最新确认的 UI 决策

以下决策以当前代码为准：

- 知音半屏卡保留当前浅色纸张/票据感，不再改成深色玻璃。
- 点击 3 个方向卡后，直接进入洱海 / 雪山 / 美食延展半屏页。
- 结果卡折叠图不再是必需 UI。
- 当前图片是阶段素材，后续替换为视频素材。

## 3. 视频信息流页

需要保持抖音视频页的熟悉结构：

- 竖屏视频背景。
- 状态栏。
- 顶部频道：上海 / 关注 / 热点 / 团购 / 商城 / 推荐。
- 左侧菜单按钮。
- 右侧互动栏：头像、关注、点赞、评论、收藏、分享、音乐盘。
- 左下作者、日期、正文、地点、音乐标签。
- 底部评论输入栏。
- 知音入口位于底部信息区附近。

知音入口不应放在：

- 屏幕中央弹窗。
- 右侧互动栏中。
- 顶部频道或搜索区旁边。
- 独立悬浮大按钮。

## 4. 知音入口

当前入口文案：

```text
你收藏的云南，有 3 种打开方式
知音已整理好
打开看看
```

入口样式：

- 半透明深色胶囊。
- 小面积知音标识。
- 不像广告弹窗。
- 与抖音底部信息层级融合。

## 5. 知音半屏卡

当前保留浅色纸张/票据感卡片。

关键 class：

- `.zhiyin-sheet`
- `.zhiyin-paper-card`
- `.paper-brand-row`
- `.understood-card`
- `.direction-card`

结构：

1. 拖拽 handle。
2. 右上关闭按钮。
3. 知音品牌行。
4. 被理解文案。
5. 3 个方向卡。
6. 当前 insight 和 evidence。
7. 轻量拨片。
8. “加一句想法”。

半屏卡首屏只解决一个问题：

```text
它为什么懂我？
```

不要在这里堆完整攻略、地图、日程或搜索筛选。

## 6. 方向卡

当前 3 个方向卡：

- 去洱海边慢下来
- 拍一组雪山人生照
- 把晚上留给本地味道

当前交互：

- 点击方向卡后直接进入对应延展半屏页。
- 不再要求先展示结果卡折叠图。

方向卡视觉：

- 小卡片横向排列。
- 图片在上，标题和摘要在下。
- 当前方向有明显选中状态。
- 卡片尺寸要保持轻量，不像攻略列表。

## 7. 延展半屏页

当前由 `CafeExploreSheet` 承载三个主题：

- `erhai`：看洱海。
- `snow`：看雪山。
- `food`：看美食。

虽然文件名仍是 Cafe，但 UI 实际已经是主题延展页。

延展页结构：

- 顶部返回。
- 居中标题。
- 右侧搜索图标仅作弱入口，不成为主交互。
- 知音继续整理的短文案。
- 横向大卡片。
- 主卡居中，两侧露出。
- 底部打开方式说明。

延展页应继续保持半屏承接感，不要变成完整攻略页。

## 8. 结果卡折叠图

历史上曾设计过三图折叠结果卡：

- `.result-card`
- `.stacked-media`

当前最新决策：

- 结果卡折叠图不再必须。
- 现阶段不需要恢复到主流程。
- 相关 CSS 可暂留，后续如长期不用再清理。

## 9. “加一句想法”

这是轻量约束入口，不是搜索框，也不是聊天输入框。

正确方式：

- 默认收起。
- 点击后出现短输入条。
- 示例：想带妈妈去 / 不想太累。
- 提交后只校准当前组合。

避免：

- 全屏搜索页。
- 聊天气泡。
- 长输入框。
- “有什么可以帮你”的助手口吻。
- 搜索结果页跳转。

## 10. 色彩与质感

信息流页：

- 视频/图片背景。
- 黑色渐变遮罩。
- 白色主文字。
- 半透明深色胶囊。

知音半屏卡：

- 当前为浅色纸张/票据感。
- 小面积使用金棕、黑色、白色层次。
- 保留轻量阴影和圆角。
- 不再要求深色玻璃风格。

延展页：

- 保持浅色半屏卡与媒体卡片组合。
- 横滑大图是视觉重心。
- 不使用大面积独立 App 式导航。

## 11. 字体与可读性

推荐字体：

```css
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif;
```

要求：

- 路演投屏时关键文案能读清。
- 信息流正文不超过 2 行。
- 方向卡标题不挤压。
- 延展页大卡片文字不遮挡关键画面。
- 替换视频素材后重新检查亮度和遮罩。

## 12. 素材状态

当前素材为阶段 `.webp` 图片。

后续替换视频时：

- 优先替换信息流背景视频。
- 再替换方向卡和延展页媒体。
- 保持 `MediaView` 统一渲染。
- 为视频准备 poster，避免首帧加载空白。

## 13. 不要做的方向

不要做成攻略页：

- Day 1 / Day 2 行程。
- 地图路线详情。
- 大量筛选器。
- 长篇说明文案。

不要做成聊天机器人：

- AI 和用户来回对话。
- 聊天气泡。
- 长输入框。
- 助手式欢迎语。

不要做成广告：

- 首屏直接卖酒店、门票、团购。
- 大面积营销 banner。
- 电商导购列表。

## 14. 路演可读性检查

每次 UI 改动后检查：

- 知音入口是否明显但不突兀。
- 半屏卡是否保留当前浅色风格。
- 3 个方向卡是否能一眼理解。
- 点击方向卡是否进入正确延展页。
- 横滑卡片是否主卡居中、两侧露出。
- 手机内 UI 是否没有文本重叠。
